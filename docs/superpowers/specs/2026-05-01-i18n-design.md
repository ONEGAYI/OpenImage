# i18n 国际化功能设计

> 日期：2026-05-01
> 状态：已批准

## 概述

为 OpenImage 添加中英文国际化支持。当前前端 105 个用户可见文本中，95 个英文 + 10 个中文混杂硬编码，无 i18n 框架。本方案引入 react-i18next 统一管理翻译，新增 LanguageSwitcher 组件供用户切换语言。

## 技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| i18n 库 | react-i18next | 业界标准，功能完善，社区支持好 |
| 默认语言 | 跟随系统语言 | 自动适配用户环境，中文环境默认中文，其他默认英文 |
| 语言持久化 | localStorage | react-i18next 内建支持，无需额外代码 |
| 翻译范围 | 仅前端 | 后端保持英文，前端根据错误类型 key 显示翻译消息 |
| 翻译文件格式 | flat key JSON | 按 `组件.功能` 前缀分组，不使用 react-i18next 命名空间功能 |

## i18n 初始化

```ts
// i18n/index.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import en from './en.json';
import zh from './zh.json';

i18n.use(initReactI18next).init({
  resources: { en: { translation: en }, zh: { translation: zh } },
  lng: localStorage.getItem('i18nextLng') ||
       (navigator.language.startsWith('zh') ? 'zh' : 'en'),
  fallbackLng: 'en',
  interpolation: { escapeValue: false },
});
```

关键点：
- JSON 直接 import，无异步加载（仅 2 种语言，体积小）
- 系统语言检测在初始化时完成，手动切换后 localStorage 优先
- `fallbackLng: 'en'` — 缺失翻译时回退英文
- 在 `main.tsx` 中 `React.render` 之前 import，确保翻译就绪

## LanguageSwitcher 组件

### 职责

纯展示 + 交互组件。通过 react-i18next 的 `useTranslation` hook 读写语言状态。

### 视觉设计

**图标**：
- 主体：简笔线条地球 SVG（经纬线圆圈），20×20 viewBox
- 右下角：2 字母语言码徽章（EN/CN），7px 字号，强调色文字（浅色 `--accent`，深色 `--accent-h`），背景跟随 `--bg`
- 按钮尺寸：34×34px，圆角 8px，与 Settings 按钮统一

**下拉菜单**：
- 向下弹出，绝对定位 `top: 100%; right: 0`，6px 间距
- 白色/深色卡片背景（`--card-bg`/`--input-bg`），1px 边框（`--border`），8px 圆角
- 三角箭头：旋转 45° 的小方块对齐按钮中心
- 每个选项：语言码（强调色，加粗）+ 语言名称 + 当前选中勾选图标
- hover 效果：浅色背景高亮

**浅色主题配色**：
- 按钮默认：`color: var(--muted)`，hover 时 `background: var(--sand); color: var(--fg)`
- 面板背景：`var(--card-bg)`，边框 `var(--border)`
- 选中项：`background: var(--surface); color: var(--fg)`
- 徽章文字：`var(--accent)`

**深色主题配色**：
- 按钮默认：`color: var(--muted)`，hover 时 `background: var(--sand); color: var(--fg)`
- 面板背景：`var(--surface)`，边框 `var(--border)`
- 选中项：`background: var(--sand); color: var(--fg)`
- 徽章文字：`var(--accent)`

### 状态管理

- 展开/收起：组件内部 `useState`
- 点击外部关闭：`useEffect` + `mousedown` 事件监听
- 语言切换：`i18n.changeLanguage(lang)`，react-i18next 自动触发全组件重渲染并持久化到 localStorage

### 语言码映射

| 检测值 | 存储值 | 菜单显示 | 徽章文字 |
|--------|--------|----------|----------|
| `en`, `en-US`, `en-GB`, ... | `en` | English | EN |
| `zh`, `zh-CN`, `zh-TW`, `zh-HK`, ... | `zh` | 简体中文 | CN |

> **注**：所有 `zh-*` 变体统一映射到简体中文。当前不支持繁体中文，若未来需要可新增 `zh-TW` 语言包。

## 组件集成

每个需翻译的组件顶部添加 `const { t } = useTranslation()`，将硬编码字符串替换为 `t('key')`。

### 集成示例

```tsx
// Before
<button title="Settings">
// After
<button title={t('topbar.settings')}>

// Before
<p>暂无会话</p>
// After
<p>{t('sidebar.noSessions')}</p>

// Before
"生成失败，请检查 API 设置或网络连接"
// After
t('error.generateFailed')
```

### 后端错误处理

前端定义常见错误的翻译 key（如 `error.network`、`error.apiKey`、`error.generateFailed`），在 catch 块中使用 `t()` 显示翻译后的消息。后端返回的原始英文错误消息作为 fallback：

```tsx
catch (err) {
  const msg = err instanceof Error ? err.message : '';
  setError(t('error.generateFailed') + (msg ? `: ${msg}` : ''));
}
```
